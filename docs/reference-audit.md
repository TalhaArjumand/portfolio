# Reference Audit

## Reel takeaway

The reel is pushing a simple idea:

- Start from a classy portfolio base
- Customize the content, theme, and sections quickly
- Ship something that looks premium enough for jobs and clients

That is useful as a direction, but it does not remove the need for:

- strong personal positioning
- good project selection
- clear call to action
- proper content structure

## Repo reviewed

- Reference repo: https://github.com/raxx21/portfolio/tree/main

## What this repo is using

- Next.js 14
- React 18
- Tailwind CSS 4
- Framer Motion

## What the current portfolio includes

- Scroll-based animated hero using image sequence frames
- Career timeline section
- Selected work section
- Footer

## What is good

- Strong first impression
- Premium dark visual style
- Clear focus on experience and projects
- Modern stack

## What is weak for our goal

- Content is hardcoded, so customization is manual
- The hero depends on a frame sequence asset pipeline
- No clear client-focused offer or services section
- No visible contact conversion flow beyond presentation
- No recruiter-focused resume / proof / CTA structure
- It is more of a developer showcase than a balanced client + job portfolio

## Decision

Do not copy this portfolio as-is.

Use it as inspiration for:

- premium feel
- motion
- bold typography
- strong project presentation

But build our own structure for:

- positioning
- conversion
- content model
- pages and sections
- easier maintenance

## Implication for our build

Our portfolio should be:

- professional
- easy to update
- strong for both recruiters and clients
- lighter and more maintainable than a frame-sequence-heavy hero

The next step is to turn your background, projects, and preferences into a proper content and layout plan.
